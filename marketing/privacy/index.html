Privacy · what is and is not true today

No overclaiming about privacy.

Most privacy pages sell a feeling. This one states the current posture, separates the marketing site from the Riot app, and is explicit about what is not yet private. For the protocol-level comparison, see the protocol field guide.

riot.protest.net (this website)

This site runs nothing behind your back.

The Riot marketing site is a dependency-free static site deployed as Cloudflare Workers assets. It has no backend, no analytics, no cookies, no remote fonts, and no third-party runtime requests. You can verify this in your browser's network tab.

What runs in your browser
  • One static HTML file per page, with inline CSS.
  • Fonts embedded as base64 data on the home page; system fonts elsewhere.
  • One inline IntersectionObserver on the home page for scroll animation — no network calls.
  • No accounts, no login, no forms, no cookies set by this site.
What this site does not do
  • No Google Analytics, Plausible, Segment, Mixpanel, or any tag manager.
  • No remote stylesheets, no remote scripts, no iframes.
  • No tracking pixels, no referrer harvesting, no fingerprinting.
  • No data passed to a third party. The only party that sees your request is Cloudflare's edge.

Cloudflare, as the Workers edge, can observe connection metadata (IP, time, requested path) as part of serving the request. That is the only third-party involvement; Riot does not add any of its own on top.

The Riot app (prototype, not shipped)

Local-first. No server round-trip for accepted data.

Riot is a community application runtime built on Willow. The design goal is that a community carries its own data and reconciles it between its own devices, with nothing accepted without a human looking at it first.

By design
  • No accounts: identity is local and signed, not issued by a server.
  • No server round-trip required to read or write data a device already holds.
  • Signed entries: authorship is verifiable without trusting any relay.
  • A community can keep using its tools locally when no server is reachable.
What sync actually means
  • Sync is reconciliation between devices a community has chosen to connect — not an automatic upload to a central store.
  • What gets carried is defined by Willow's data model; see willow-architecture.md for the precise mapping.
  • The app itself runs no telemetry; there is no "phone home" path in the codebase.

What Riot does not provide yet

Honest boundaries — do not rely on what is not shipped.

Riot will not borrow privacy language from systems that have features Riot has not shipped. The boundaries below are the same ones stated in the protocol field guide, restated here so they are easy to find.

Not confidential today
  • Current public spaces are not confidential. Treat anything in a public Riot space as publishable.
  • Private encrypted groups are not shipped. They are being reviewed under their own threat model before anything ships.
  • Anyone operating a relay can read what that relay stores. Local-first removes a mandatory home server; it does not encrypt the relay by default.
Offline does not mean invisible
  • Local-first does not repeal radio range, device loss, or hostile peers. A seized or compromised device still holds whatever it holds.
  • Out-of-band identity verification (a QR code, a portable invite file) is still human work. Riot does not solve trust by itself.
  • Riot is not an audited, hardened tool. It is a prototype. Use established encrypted messengers for anything that must stay secret today.

If a feature is not listed on this page as shipped, assume it is not. For how Riot compares to Briar, Matrix, Bitchat, and others on the privacy boundary specifically, see the protocol profiles.
