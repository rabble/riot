# Compact Tool Breadcrumb Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the oversized macOS tool close row with a compact, responsive, tappable Community › App › Page breadcrumb that makes location and escape routes obvious without changing iPhone navigation.

**Architecture:** `AppRuntimeView` owns the mounted tool's app/page chrome and WebKit title bridge, while `CommunityShellView` supplies community and routing actions. The community chooser receives explicit mounted-tool context so current-community selection is harmless and cross-community or Nearby transitions cannot leave a hidden live WebView. Wiki publishes a small host contract through `document.title` and a fixed `riot:navigate-root` window event.

**Tech Stack:** SwiftUI, WebKit/KVO, XCTest, JavaScript, Rust starter-app packer, Xcode project files.

```swift
enum AppBreadcrumbTitle {
    static func page(from rawTitle: String?, appName: String) -> String? {
        guard let rawTitle, rawTitle.utf16.count <= 512 else { return nil }
        guard rawTitle.unicodeScalars.allSatisfy({ scalar in
            switch scalar.properties.generalCategory {
            case .control, .format, .lineSeparator, .paragraphSeparator: return false
            default: return true
            }
        }) else { return nil }
        let normalized = rawTitle.precomposedStringWithCanonicalMapping
            .trimmingCharacters(in: .whitespacesAndNewlines)
        let suffix = " — \(appName)"
        guard normalized.hasSuffix(suffix) else { return nil }
        let page = String(normalized.dropLast(suffix.count))
            .trimmingCharacters(in: .whitespacesAndNewlines)
        guard !page.isEmpty, page.count <= 120 else { return nil }
        return page
    }
}

struct AppBreadcrumbLabels: Equatable {
    let community: String
    let app: String
    let page: String?
    var full: [String] { [community, app, page].compactMap { $0 } }
    var compact: [String] { page == nil ? ["🏘", "🧰"] : ["🏘", "🧰", "📄"] }
}
```

Add a macOS-only `AppBreadcrumbView` whose full candidate is a 36-point row of plain buttons separated by decorative `›` labels. Put the full and emoji candidates in `ViewThatFits(in: .horizontal)`, apply `.fixedSize(horizontal: true, vertical: false)` to each candidate, give every emoji button the same full accessibility label and `.help(...)` as its text equivalent, and expose identifiers `breadcrumb-community`, `breadcrumb-app`, and `breadcrumb-page`. - [ ] **Step 3: Implement chooser decisions and shell routing**

Add:

```swift
enum CommunityChooserSelectionDecision: Equatable {
    case dismissCurrent, confirmSwitch, switchImmediately

    static func decide(selectedID: String, currentID: String?, mountedAppName: String?) -> Self {
        if selectedID == currentID { return .dismissCurrent }
        return mountedAppName == nil ? .switchImmediately : .confirmSwitch
    }
}

enum ToolRoutePolicy {
    static var closesMountedToolBeforeRoute: Bool {
        #if os(macOS)
        true
        #else
        false
        #endif
    }
}

enum MountedToolExit {
    static func perform(closeTool: () -> Void, then action: () -> Void) {
        closeTool()
        action()
    }
}

enum CommunityChooserConfirmation: Equatable {
    case stay, switchCommunity

    func perform(closeTool: () -> Void, switchCommunity: () -> Void) {
        guard self == .switchCommunity else { return }
        MountedToolExit.perform(closeTool: closeTool, then: switchCommunity)
    }
}
```
