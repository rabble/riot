Riot is an offline‑first native app for creating, verifying, and carrying information when the internet is shut down, censored, protested, or simply gone. Preload it before the crisis. Publish, sync, and merge from inside it during the crisis.

What it is

Two subsystems, one bridge

Riot splits into two parallel subsystems joined only by an explicit, signed, human‑initiated bridge — never an automatic copy. Lineage: indymedia.org's open publishing, protest.net's structured events, TxtMob's broadcast alerts, divine.video's syndicated media. The through‑line is publishing infrastructure, not chat.

Public / plaintext

Open Newswire

Per‑incident open spaces anyone can post to, and publication spaces where a pseudonymous collective is the publisher. Subscribers' devices are the distribution network — a banned publication has no server to seize.

  • Open space — communal namespace, frictionless publishing, no accounts
  • Publication space — owned namespace, capability‑gated writers, collective identity
  • Curation — a reading lens applied at read time, never a gate on writing
  • Packets — alerts, resource maps, legal guides, medic locations, rumor corrections
Private / encrypted

Private Groups

Encrypted, unlinkable Willow namespaces for affinity groups, coops, and crews — joined in person via QR/NFC or by portable encrypted invite files.

  • Control plane — MLS orders membership epochs, add/remove/recovery
  • Data plane — ordinary Willow entries inside an opaque, padded drop
  • Identity — unlinked personas; a group key is never a newswire key
  • Panic — per‑group and full‑device wipe, keys destroyed first
Bridge Content crosses only as an explicit, signed user act. Group → newswire strips every private identifier and keeps the AI‑assistance flag. Newswire → group keeps the original public entry fully intact and adds a private annotation. Never automatic, never a shared store.

How it holds together

Willow underneath everything

Willow is the canonical data model: signed, path‑addressed, mergeable entries. A packet, incident, or community maps to a namespace; a signer, site, or device to a subspace; a page or object to a path. Stores merge through newer‑write replacement and prefix pruning — a better fit for evolving field information than append‑only chat.

Exchange ladder

A staged build order: each rung only activates after the one before it clears its own evidence gate.

PHASE 0

Evidence bundle

Prove canonical Willow entry/capability bytes, Meadowcap authority, bounded preview‑first import, and a two‑way native handoff. Not yet interoperable by design.

PHASE 1

Drops

Willow Drop Format for whole‑space, selective, or delta export. USB, SD, AirDrop, email — any channel that moves a bytestring.

PHASE 2

Live sync

Willow Transfer Protocol over local Wi‑Fi/LAN once drop import/export is solid. Request/response, no live negotiation required first.

PHASE 3

Confidential sync

Private interest overlap and partial sync — only once the app actually needs it. More power, much more complexity.

Ten durable wire kinds

alert
Urgent instruction — urgency, severity, area, required expiry
observation
A signer's bounded statement of what, where, when, how
event
Scheduled activity — start, end, status, location
resource
A service, facility, or supply — availability & location
request
A need for an item/service — the UI label is "need"
offer
Available capacity with constraints and validity
commitment
A signed link between a request and an offer
task
Dispatchable work — open → claimed → done, signed each step
document
Bulletin, guide, checklist, runbook
annotation
Correction, verification, moderation, translation, fulfillment

Safety principles

  • Preview before ingest — every import is manifest, signers, and size shown before anything lands.
  • Trust as a lens, not a gate — open publishing stays frictionless; curation and verification happen at read time.
  • Required expiry for every operational update — alerts and needs cannot go stale silently.
  • Panic wipe — per‑group and full‑device, keys destroyed before data.
  • No network beacons from rendered packet content, and no arbitrary native bridge from web content.
  • Local‑only by default — no accounts, no canonical server, no carrier chokepoint.
  • The LLM drafts, never publishes — a human reviews and signs every update; machine help is always disclosed.
  • Provenance survives re‑export — signer, source, and import history travel with the content.

Anchored in evidence

What people actually need

Design decisions here are pulled from adversarially verified field research, not assumption — in the spirit of New_Public's Front Porch Forum study: close, evidence‑based accounts of what real people needed from a real platform in a real crisis, not abstract workflow diagrams. Each pass runs a multi‑source deep‑research workflow — search, fetch, extract falsifiable claims, then put every claim in front of three independent adversarial verifiers who default to refuting it.

DISPATCH 01 · 2026‑07‑10

How mutual aid & grassroots networks actually coordinate

23sources
115claims
24confirmed
1refuted

Across Occupy Sandy, Verificado 19S, TXTMob, Indymedia, and NYC COVID mutual aid, grassroots networks converge on one pipeline: intake → verification by trusted roles → dispatch to fulfillment, with a persistent split between field roles and remote coordinators who broadcast only verified information.

3‑0 ✓A shared editable ledger — Occupy Sandy's master spreadsheet, COVID mutual aid's Airtable — surrounded by paper artifacts as both intake instrument and degraded‑mode transport.
3‑0 ✓TXTMob formalized a 2×2 channel matrix — public/private × moderated/unmoderated — mapping to broadcast, dispatch, open forum, and affinity‑group channels.
3‑0 ✓Indymedia UK's default moderation was hide, not delete — hidden posts stayed publicly inspectable; governance lived on a separate meta‑channel.
3‑0 ✓NYC COVID groups ran on novel configurations of consumer tools — Slack, Airtable, Google Forms — with analog flyers as a required complement, not a fallback.
For Riot Directly grounds the wire vocabulary: task, verification, and moderation_action as first‑class object types; roles (intake, dispatcher, field verifier, moderator) as named Meadowcap capability bundles; a governance path kept separate from content, mirroring Indymedia's rule that disputes stay off the newswire.

DISPATCH 02 · 2026‑07‑11

What people actually need — disaster, protest, mutual aid

23sources
96claims
18confirmed
7refuted

Closing four open questions from Dispatch 01. Non‑crisis mutual aid is blocked by psychological friction, not missing features. Real infrastructure collapse (Turkey‑Syria 2023, Hurricane Maria 2017) shows consumer chat apps remain the default even for trained responders, and infiltration defense is social, not technical.

3‑0 ✓Timebank members act on intent to ask for help far more than intent to offer it — an asymmetry, not a symmetric UX problem.
3‑0 ✓Even trained international SAR teams fell back to WhatsApp after the 2023 Türkiye earthquakes; satellite phones were scarce and externally supplied.
3‑0 ✓Ham radio after Hurricane Maria was an activated institutional partnership (Red Cross → ARRL), not ad‑hoc hobbyist relay.
2‑1 ✓UK climate activists vet infiltrators in person — and visibly adopting a maximally secure tool can itself be read as a suspicion signal.
For Riot Don't build symmetric request/offer UX — lower the barrier to asking. Let private groups calibrate their own visible security posture instead of forcing one default. Treat radio/satellite relay as an externally activated institutional resource, not something the app can assume or auto‑recruit.

DISPATCH 03 · 2026‑07‑11

Hybrid local gossip + opportunistic internet backhaul

24sources
90claims
21confirmed
4refuted

Riot shouldn't assume a binary online/offline world. Real systems that let some nodes bridge to the wider internet while most stay local‑only converge on one pattern: keep merge logic transport‑agnostic, and make bridge selection cheap and untrusted.

3‑0 ✓Briar's sync protocol is transport‑agnostic — identical causal‑DAG delivery over Bluetooth, Wi‑Fi, memory cards, or Tor, inside the same audited app.
3‑0 ✓Secure Scuttlebutt pub servers are ordinary, untrusted peers with a static IP — feeds are self‑verifying, so a relay can withhold data but never forge it.
3‑0 ✓Meshtastic's mesh‑to‑MQTT gateway is fully opt‑in and unauthenticated — but bridge confidentiality was not confirmed; brokers can plausibly see plaintext.
0‑3 ✗Academic DTN/PRoPHET gateway‑selection heuristics remain simulation‑only — no production deployment found.
For Riot Give local sync and the public gateway one shared reconciliation primitive so the gateway is just another opportunistic peer. Make bridge selection opt‑in and unauthenticated. Confidentiality must never depend on trusting the bridge — sign everything, encrypt what's sensitive, independent of who relays it.

DISPATCH 04 · 2026‑07‑11

Distribution with no live connectivity at all

22sources
105claims
24confirmed
1refuted

RSF's CDN‑mirroring "Collateral Freedom" assumes the reader has some working internet. Riot's real scenario has none. Physical sneakernets have real, sustained, multi‑year deployed evidence; app‑layer BLE mesh does not.

3‑0 ✓Cuba's El Paquete Semanal — a hierarchical ~1TB weekly package moved by car, plane, train, bus — is the strongest deployed evidence for a package‑plus‑courier sync model.
3‑0 ✓DakNet's bus‑mounted data mules connected village kiosks in Cambodia and India at two orders of magnitude lower cost than landlines.
3‑0 ✓Bridgefy shipped with no authentication and a whole‑network DoS from one crafted message; Hong Kong 2019 protesters said "it just doesn't work" and fell back to paid SMS.
3‑0 ✓Briar's one audit finding was in its own offline app‑sharing feature — a physical‑proximity PSK brute‑force, not the online protocol.
For Riot Prioritize a hierarchical physical‑media/data‑mule sync path as first‑class, not fallback — Willow Drop Format is already built for USB/SD/courier transport. Any offline pairing or install mechanism is its own attack surface and needs long, high‑entropy secrets, not "offline implies safe."

DISPATCH 05 · 2026‑07‑11

Arti as Riot's Tor backhaul

25sources
100claims
19confirmed
6refuted

Briar bundles the C tor daemon. Riot's core is Rust. Can Arti — the Tor Project's Rust reimplementation — embed directly instead?

3‑0 ✓Arti's own docs name in‑Rust embedding via arti_client as the primary integration path — async streams, no separate daemon needed in principle.
3‑0 ✓Arti ships no UniFFI or mobile bindings — its own iOS guide requires hand‑rolled FFI and admits it "hasn't been tested by many people."
3‑0 ✓Client + onion‑service hosting is mature since 2022; the inbound relay/OR‑port listener was still under active construction as of Dec 2025.
3‑0 ✓Cure53's only High‑severity finding across an 18‑issue mobile‑tunnel audit was in the Apple/Android FFI boundary code — not core Tor logic.
For Riot Near‑term: outbound onion‑service hosting over Arti, not inbound relay hosting. Treat the UniFFI/FFI boundary as the highest‑risk surface in the whole integration and get it independently reviewed — the one severe bug found anywhere near Arti was exactly there.

DISPATCH 06 · 2026‑07‑11

Optional media authenticity: ProofMode & C2PA

26sources
114claims
18confirmed
7refuted

C2PA/ProofMode provenance is built to disclose capture metadata as proof of authenticity. Riot's safety model needs the opposite for protest photography — strip that same metadata before publish. Does the tension actually resolve?

3‑0 ✓ProofMode's proof bundle is already detachable from the media file — original photo, hash, sensor CSV, signature, kept separate, not baked in.
3‑0 ✓C2PA formally supports redacting assertions like GPS, not restricted to the original signer — a downstream publisher can strip them before release.
3‑0 ✓The same "exclusion range" that enables redaction leaves those bytes unprotected — anyone can alter or insert false data there without breaking the signature.
3‑0 ✓ProofMode's persistent per‑install signing key contradicts its own no‑persistent‑identity design goal — a real repeat‑contributor correlation risk.
For Riot Model this as a detachable verification annotation, redacted by default before anything crosses the Group→Newswire bridge, full provenance kept private. Record what was redacted — a "valid" check doesn't cover excluded fields.
